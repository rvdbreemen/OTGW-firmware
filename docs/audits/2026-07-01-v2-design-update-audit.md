# v2 Web UI vs updated design mockup — audit (2026-07-01)

Design project "Mockups into design system" (`eb26ec05-912a-40f0-97cc-d63e770218e0`) updated the
ground-truth prototype `docs/design/boiler-dashboard-concepts.html` (+530/-45 lines vs the 2026-06-26
import that TASK-933 aligned against). This audit compares the CURRENT v2 implementation
(`src/OTGW-firmware/data/v2.{html,js,css}`) against the updated mockup. Method: 7 parallel area
audits, each finding adversarially verified against the sources (workflow `wf_9d378fc8-1c6`,
14 agents). Verdicts: 37/38 confirmed, 1 corrected.

States: `missing` = mockup feature absent in v2 · `partial` = some of it exists · `divergent` = exists, different shape · `present` = already matches.

## Header — identity chips + status badges

The v2 header is still the pre-redesign minimal bar (title + clock + Classic-UI/theme toggles); the entire "firmware-authentic header identity + status badges" block from the updated mockup is unimplemented: no identity chips, no Connected/mode status pill, no SIMULATION badge, no hdr-spacer layout, no fw-badge sync in renderConnStrip. All required data (hostname, fwversion, ipaddress, ssid, wifirssi, wifiquality, otgwmode, WS state, simulation active) is already exposed by the firmware via /api/v2/device/info, /api/v2/health and /api/v2/simulate, so this is pure UI work with no firmware changes strictly required.
