# 2.0.0 Beta — hardware-validatie checklist

Status: opgesteld 2026-06-28 tijdens de beta-readiness review. Dit zijn de items
die **niet** vanaf de host te sluiten zijn: ze hebben een echte OTGW32 (en in
sommige gevallen een live OT-bus / HA-broker) nodig. Alle software-zijde is
geland; dit is wat een mens op hardware moet bevestigen.

## Blocker — eerst doen

### TASK-940 — SoftAP fallback bereikbaar na STA-verlies
- **Wat:** fix geland (passphrase `otgw123` → `otgw1234`, ≥8 chars, alpha.279).
- **Test:** flash een build met de fix, forceer STA-verlies (verkeerd WiFi-ww of
  router uit), bevestig dat de fallback-AP opkomt en dat `http://192.168.4.1`
  het config-portal serveert.
- **AC-gate:** TASK-940 AC#2.
- **Waarom blocker:** dit is het verschil tussen "tester power-cyclet en herstelt"
  en "tester moet herflashen". Raakt elke nieuwe beta-tester die WiFi verliest.

## Hoog — eerste beta-soak

### TASK-879 / ADR-149 — pcb-plafond realistische soak
- **Wat:** de loop-blokkerende bugs zijn in code opgelost; het resterende
  faalpad (LWIP pcb-uitputting onder adversariële concurrency) is geaccepteerd
  als hardware-plafond met automatische TWDT-recovery.
- **Test (realistisch, NIET de flood):** 1 live-log-tab open + HA MQTT actief +
  normaal browsen, ≥1 uur, op een echte OTGW32. Bevestig **bootcount delta 0**.
- **AC-gate:** TASK-879 AC#4.
- **Release-note:** documenteer het pcb-plafond (adversariële flood → reboot →
  auto-recovery) zodat testers het verwachten.

## Veld-bevestiging — nieuwe HA-features (deze run)

### TASK-943 — heating-source manueel-autoritatief (deze run, alpha.281)
- **Wat:** auto-detect gedemoteerd tot niet-control hint; AUTO → veilige
  gas-ketel default voor control; detectie leest nu de juiste MsgID 3 cooling-bit.
- **Test:** op een SAT-device met `satsource=AUTO`: bevestig dat control zich als
  gasketel gedraagt (max-setpoint cap niet 40C tenzij handmatig HEAT_PUMP gekozen).
  `heating_source_detected` mag de cooling-hint tonen zonder control te sturen.

### TASK-942 — hvac_mode / hvac_action als HA-sensors (deze run, alpha.282)
- **Wat:** twee standalone discoverable HA-sensors op de bestaande topics.
- **Test:** op een echte OTGW32 met HA: bevestig dat beide sensors in discovery
  verschijnen (onder het OT-Core device) en juiste waarden tonen over off/heat/cool.
- **AC-gate:** TASK-942 AC#6.

### TASK-939 — unified heat/cool/off climate (al geland, alpha.279)
- **Test:** jelvank veld-sign-off op cooling (1.x-twin TASK-938 al bevestigd).
