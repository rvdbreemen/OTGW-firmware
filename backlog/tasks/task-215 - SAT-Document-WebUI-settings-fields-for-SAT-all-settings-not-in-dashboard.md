## Description

<!-- SECTION:DESCRIPTION:BEGIN -->
The SAT settings page (displaySATSettingsPage in index.html) is dynamically generated by sat.js. It is not documented anywhere. A document listing all SAT settings fields that appear in the WebUI is needed, covering:\n\n- Field label, input type, valid range, and default value for each setting\n- Which settings are in the dashboard view vs. the settings page\n- Which settings are only accessible via MQTT/REST (no WebUI field)\n\nThe firmware has 65+ SAT settings (all keys starting with SAT in settingStuff.ino lines 279-354). The WebUI settings page content is rendered from sat.js. An audit of sat.js is needed to list which settings have UI fields.\n\nKey settings likely missing from WebUI (requiring MQTT/REST to configure):\n- SATsensormaxage (sensor max age, 21600s default)\n- SATerrormon (error monitoring)\n- SATautogains (auto-gains value, 2.0)\n- SATheatingmode (0=comfort, 1=eco)\n- SATcyclesperhour (2-6, default 3)\n- SATvalveoffset (-1 to 1, default 0.0)\n- SATsolarfreezeint (freeze integral during solar gain)\n- SATwindowminsec (window min open seconds, 60s)\n- SATsimheatrate / SATsimcoolrate (simulation rates)\n\nSource: src/OTGW-firmware/data/sat.js, src/OTGW-firmware/settingStuff.ino (lines 279-354), src/OTGW-firmware/OTGW-firmware.h (struct SATSection lines 737-821)."
