# Handoff to Claude Code — OTGW design system integration

This package was generated by the design pipeline. The Windows script
`apply.ps1` has copied (or will copy) the new files into
`src/OTGW-firmware/data/`. **Your job from here:** review each change
in the working tree, make the small edits the script can't make
automatically, and commit per logical patch.

You are reviewing **before** anything ships. Treat each step as a PR
you would block if it's not right.

## What's already on disk after `apply.ps1`

New files in `src/OTGW-firmware/data/`:

- `ds-tokens.css` — all design tokens (colors, type, spacing, radii, shadows, motion). Light + dark.
- `components.css` — full component library, token-driven.
- `theme-toggle.js` — `☀ / ☾` toggle, sets `body.dark` + `[data-theme]`, persists to localStorage, dispatches `theme:changed`.
- `sat-slider.js` — live-fill for `.ds-slider` ranges, idempotent, listens for `sat:rendered`.
- `echarts-theme.js` — `otgwChartTheme()` reads CSS vars so charts follow tokens.
- `design.html` — component reference page. Visit `/design.html` on the device for visual regression.
- `fonts/inter-400.woff2`, `fonts/inter-700.woff2`, `fonts/jetbrains-mono-400.woff2`.

Existing files **not touched by the script**: `index.html`,
`index.css`, `index_dark.css`, `sat.html`, `sat.js`, `settings.html`,
`FSexplorer.html`, `FSexplorer.css`, `graph.js`, etc.

   ```js
   if      (state === 'enabled')  { pill.textContent = 'Enabled';  pill.classList.add('is-ok'); }
   else if (state === 'learning') { pill.textContent = 'Learning'; pill.classList.add('is-warn'); }

   ```
3. ECharts init — wrap with `otgwChartTheme()`:
   ```js
   const chart = echarts.init(document.getElementById('sat-chart'));
   function paint() {

       xAxis: { type: 'time' },
       yAxis: { type: 'value', name: '°C' },
       series: [{ type: 'line', data: HISTORY, smooth: true }],
     }), true);
   }
   paint();
   document.addEventListener('theme:changed', paint);
   window.addEventListener('resize', () => chart.resize());
   ```

## Smoke test (pre-merge)

After flashing the test device:

1. `curl -I http://<device>/ds-tokens.css` → 200, ~8 KB.
2. `curl -I http://<device>/design.html` → 200. Open it in a browser,
   click `☾` — entire page swaps theme. Every component renders in
   both modes without contrast issues.
3. Home page: cards neutral white / `#2c2c2e`, no light-blue tint.
4. SAT page: target tile is the only blue accent; DHW slider fills
   live; chart axis colors are visible in dark mode.
5. Settings: form rows render; sticky save bar visible (if applied).
6. FSexplorer: file list zebra-striped; system action buttons in
   their semantic colors.
7. Log viewer: terminal panel is `#1e1e1e` in both themes
   (intentional — it's a self-contained terminal).
8. DevTools console — no 404s, no CSS warnings.

## Drift gate for future package work

TASK-470 / ADR-091 promote the design-system class drift detector into
`evaluate.py`. Any future package output that emits a class from HTML or
JavaScript must either define that class in `data/*.css` or add it to the
documented allowlist in `evaluate.py` with a concrete no-style reason.

## Rollback

The package's `apply.ps1` writes a timestamped zip in
`<package>/backups/data-backup-<stamp>.zip`. Unzip it over
`src/OTGW-firmware/data/` to restore the pre-patch state. Or just
`git checkout HEAD -- src/OTGW-firmware/data` if you've not committed.

## Things explicitly out of scope for you

- Don't refactor `graph.js` / `index.js` beyond the per-patch edits.
- Don't bump the firmware version; the design layer is non-functional.
- Don't migrate the OpenTherm protocol layer.
